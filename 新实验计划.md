## 目标

在新的设置下重新组织 baseline、AER 超参数搜索、主实验和完整评测，确保所有实验可复现、可对比，并能支撑论文中的正确性与多样性分析。

## 统一设置

### 固定训练设置

- 基础训练算法：GRPO。
- baseline 形式：naive GRPO，不加入任何探索奖励，即 `tau = 0`。
- baseline 最少训练步数：`500` steps。
- 熵正则 baseline：熵正则项中的熵由“求和”改为“求平均”。
- 所有实验应尽量保持模型、数据、采样配置、batch size、学习率、KL 设置、rollout 数量、随机种子等一致。

### 可配置超参数

- `similarity_algorithms`：训练过程中需要计算探索奖励的相似度算法列表。
- `gamma_list`：由 tau 校准得到候选 tau 的倍率列表，默认建议：
  - `1.1`
  - `1.2`
  - `1.3`
  - `1.4`
- `calibration_steps`：用于统计各算法探索奖励最小值的 baseline 步数，至少覆盖 baseline 的 `500` steps。
- `target_similarity_for_gamma_search`：优先用于搜索 gamma 的相似度算法。
- `main_similarity_algorithms`：主实验需要跑的相似度算法，优先级：
  - `token_match`
  - `ngram_overlap`
  - `semantic_embedding`
  - 其他相似度算法视时间继续补充。

## 阶段 1：tau=0 baseline 校准实验

### 实验 1.1：naive GRPO baseline

目的：

- 跑一次不加探索奖励的 naive GRPO baseline。
- 在训练过程中每一步都计算所有配置的相似度算法对应的探索奖励。
- 这些探索奖励只用于记录和校准，不参与训练目标。

配置：

- `tau = 0`
- `use_exploration_reward = false`
- `similarity_algorithms = 可配置列表`
- `train_steps >= 500`

需要记录：

- 每一步的原始任务奖励。
- 每一步各相似度算法对应的探索奖励。
- 每个相似度算法在整个 baseline 训练过程中的探索奖励最小值：
  - `min_exploration_reward[token_match]`
  - `min_exploration_reward[ngram_overlap]`
  - `min_exploration_reward[semantic_embedding]`
  - 其他算法同理。

产物：

- baseline checkpoint。
- baseline 训练日志。
- 各算法探索奖励统计表。
- 后续 tau 计划表。

### 实验 1.2：熵正则 baseline

目的：

- 在 baseline 设置下加入熵正则。
- 修改熵正则求熵方式：由 token 或样本维度上的求和改为求平均。

配置：

- `tau = 0`
- `use_exploration_reward = false`
- `use_entropy_regularization = true`
- `entropy_reduction = mean`

需要确认：

- 熵正则项实际使用的是平均熵。
- 与旧的求和版本相比，loss 数值尺度发生变化后，训练没有异常发散。

产物：

- entropy baseline checkpoint。
- entropy baseline 训练日志。
- 与 naive baseline 的基础指标对比。

## 阶段 2：由 baseline 统计结果生成 tau

对每个相似度算法，根据阶段 1.1 中记录到的探索奖励最小值生成候选 tau：

```text
tau[algorithm, gamma] = min_exploration_reward[algorithm] * gamma
```

默认候选倍率：

```text
gamma_list = [1.1, 1.2, 1.3, 1.4]
```

需要生成的表：

| similarity algorithm | min exploration reward | gamma | tau |
| --- | ---: | ---: | ---: |
| token_match | 待填 | 1.1 | 待填 |
| token_match | 待填 | 1.2 | 待填 |
| token_match | 待填 | 1.3 | 待填 |
| token_match | 待填 | 1.4 | 待填 |
| ngram_overlap | 待填 | 1.1 | 待填 |
| ngram_overlap | 待填 | 1.2 | 待填 |
| ngram_overlap | 待填 | 1.3 | 待填 |
| ngram_overlap | 待填 | 1.4 | 待填 |
| semantic_embedding | 待填 | 1.1 | 待填 |
| semantic_embedding | 待填 | 1.2 | 待填 |
| semantic_embedding | 待填 | 1.3 | 待填 |
| semantic_embedding | 待填 | 1.4 | 待填 |

注意：

- tau 的计算必须来自同一次 tau=0 baseline 的日志，避免不同训练轨迹带来的统计偏差。
- 如果探索奖励可能出现负数，需要确认 tau 在实现中的语义是否允许负值；若不允许，应额外记录最小正值或分位数值，并在实验记录中说明。

## 阶段 3：单个相似度算法的 gamma 搜索

目的：

- 先选定一个相似度算法，搜索合适的 `gamma`。
- 只改变 `gamma/tau`，其他训练配置保持和 baseline 一致。

建议优先搜索算法：

- 首选：`ngram_overlap`
- 如已有更稳定或更快的实现，也可改为 `token_match`。

实验矩阵：

| experiment | similarity algorithm | gamma | tau 来源 | 说明 |
| --- | --- | ---: | --- | --- |
| gamma-search-1 | 选定算法 | 1.1 | baseline min reward * 1.1 | AER 训练 |
| gamma-search-2 | 选定算法 | 1.2 | baseline min reward * 1.2 | AER 训练 |
| gamma-search-3 | 选定算法 | 1.3 | baseline min reward * 1.3 | AER 训练 |
| gamma-search-4 | 选定算法 | 1.4 | baseline min reward * 1.4 | AER 训练 |

选择最佳 gamma 的依据：

- 主指标优先看正确性：
  - average accuracy
  - first@1
  - pass@1/2/4/8/16/32/64/128
- 同时检查多样性是否提升或至少没有严重下降：
  - distinct_2
  - self_bleu4
  - equational diversity
- 若正确性接近，优先选择多样性更好的 gamma。
- 若多样性明显提升但正确性明显下降，应记录为 trade-off，不直接作为主实验默认 gamma。

产物：

- 每个 gamma 的训练日志和 checkpoint。
- 每个 gamma 的评测结果。
- 最佳 `gamma_best` 和对应 `tau_best`。

## 阶段 4：复用最佳 gamma 运行主 AER 实验

目的：

- 使用阶段 3 选出的 `gamma_best`。
- 对剩余相似度算法复用相同的 gamma 策略运行 AER 主实验。

tau 计算：

```text
tau[algorithm] = min_exploration_reward[algorithm] * gamma_best
```

优先实验：

| priority | experiment | similarity algorithm | gamma | tau 来源 |
| ---: | --- | --- | ---: | --- |
| 1 | aer-token_match | token_match | gamma_best | baseline min reward * gamma_best |
| 2 | aer-ngram_overlap | ngram_overlap | gamma_best | baseline min reward * gamma_best |
| 3 | aer-semantic_embedding | semantic_embedding | gamma_best | baseline min reward * gamma_best |
| 4 | aer-other | 其他算法 | gamma_best | baseline min reward * gamma_best |

需要保证：

- 每个主实验都与 baseline 使用相同训练步数或相同训练预算。
- 每个主实验都保存可用于完整评测的 checkpoint。
- 每个主实验都记录训练过程中的任务奖励、探索奖励、总奖励和关键 loss 项。

## 推荐执行顺序

1. 修改并确认熵正则实现：熵由求和改为求平均。
2. 跑 naive GRPO tau=0 baseline，至少 500 steps，并在每一步记录所有相似度算法的探索奖励。
3. 从 baseline 日志中统计每个算法的探索奖励最小值。
4. 根据 `gamma_list` 生成每个算法的 tau 候选表。
5. 选择一个算法进行 gamma 搜索。
6. 对 gamma 搜索实验做完整评测，确定 `gamma_best`。
7. 使用 `gamma_best` 对 `token_match`、`ngram_overlap`、`semantic_embedding` 跑主 AER 实验。
8. 时间允许时继续补其他相似度算法。
9. 对 baseline 和所有主 AER 实验运行完整评测。
10. 汇总正确性、多样性、训练稳定性和计算开销。

## 最低完成标准

必须完成：

- naive GRPO baseline，`tau = 0`，训练不少于 500 steps。
- baseline 训练过程中每一步记录所有目标相似度算法的探索奖励。
- 熵正则 mean 版本 baseline。
- 至少一个相似度算法的完整 gamma 搜索。
- `token_match`、`ngram_overlap`、`semantic_embedding` 三个主 AER 实验。
- 所有主实验的完整正确性和多样性评测。

可选补充：

- 更多相似度算法。
- 更多 gamma 值。
- 多随机种子复现实验。
- 训练曲线、tau 敏感性曲线、多样性-正确性 trade-off 图。

## 最终需要整理到论文中的结论维度

- AER 是否提升 average accuracy 和 pass@k。
- AER 是否提升多样性指标。
- 不同相似度算法的收益差异。
- gamma/tau 对训练结果的敏感性。
- 熵正则 mean baseline 与 naive baseline 的差异。
- 正确性提升与多样性提升之间是否存在 trade-off。
