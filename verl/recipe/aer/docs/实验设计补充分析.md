# 实验设计补充分析

## 1. 训练步数

原先 `train_batch_size=32` 时 300-400 step 收敛，对应约 `9600-12800` 个 prompt 更新样本。现在 `train_batch_size=128`，若只按样本数粗略等价，收敛区间可能变成 `75-100` step。但 RL 训练还受 optimizer step 数、数据分布、rollout 方差影响，不能直接把步数除以 4。

因此采用 naive GRPO bs=128 收敛扫描确定统一 `S_final`。`S_final` 设下限 120 是为了保证足够 optimizer 更新；上限先设 240 是为了避免过度消耗 4 张 H20。若 240 step 仍未平台化，再扩展到 320。

## 2. save_freq 与 checkpoint 保留

`test_freq=12` 时每 12 step 有一次验证，`save_freq=24` 表示每两个验证点保存一次 checkpoint。这样可以覆盖性能峰值附近的模型，又不会像每次验证都保存那样占用过多存储。

推荐：

```bash
save_freq=24
max_actor_ckpt_to_keep=6
max_critic_ckpt_to_keep=6
```

最多保留最近 6 个 checkpoint，覆盖约 144 step 的窗口。若 `S_final=120`，基本保留全程关键 checkpoint；若 `S_final=240/320`，保留后期最可能成为最终候选的 checkpoint。FSDP checkpoint 默认不是 HuggingFace 格式，最终做重新推理评测前再用 `scripts/model_merger.py` 合并目标 checkpoint，避免每次保存都写一份完整 HF 模型。

## 3. tau 取值

AER 动态权重为：

```text
w_{t+1} = clip(w_t + (tau - r_exp_bar_t), 0, 1)
```

若 tau 低于校准训练中后期探索奖励，权重长期接近 0，AER 退化为 naive GRPO；若 tau 过高，权重过早饱和到 1，最终 reward 易被探索奖励主导。

本设计将 `tau=0` 校准实验前 72 step 均分为三段：`step 1-24`、`step 25-48`、`step 49-72`。三段分别代表早期、中期、后期探索奖励水平，不再额外计算 `tau_anchor`。

三档 tau 分别为三段 `metric/exploration reward` 均值，并保留三位小数。对当前 token_match naive GRPO 日志，后续 token_match 实验使用 `tau_1=0.638`、`tau_2=0.591`、`tau_3=0.552`。这不是最终只选一个跑长程，而是每种相似度算法至少 3 个 tau 都跑到 `S_final`。若 3 个预设 tau 都不超过 baseline，应先记录为预设网格下的负结果；后续可以根据权重曲线追加更低或更高 tau 做诊断，但追加实验需要标记为探索性结果。

## 4. 资源量

最少训练数量：

| 类型 | 数量 |
|---|---:|
| naive GRPO 收敛扫描 | 1 |
| entropy-GRPO 正式实验 | 3 |
| 非 token_match 校准 | 4 |
| AER 正式实验 | 15 |

最少 23 次训练，不含 tau 追加与多 seed。优先级：

1. naive GRPO 收敛扫描，确定 `S_final`。
2. token_match 三档 AER，验证核心算法是否成立。
3. entropy-GRPO 三档 baseline。
4. ngram_overlap、levenshtein、simhash。
5. semantic_embedding。

semantic_embedding 成本最高，但能补充语义相似度视角，因此保留为完整性实验。

## 5. 指标依据

Pass@k 使用 repeated sampling 下“至少一个样本正确”的估计方式，适合评估采样多样性带来的可解题概率；Pass@1 仍作为主性能指标。Distinct-2 衡量词面 2-gram 多样性，Self-BLEU 衡量同题多输出之间的重复度，Semantic Cosine 用 embedding 空间相似度补充词面指标。

参考资料：

- Pass@k：Chen et al., 2021, Evaluating Large Language Models Trained on Code, https://arxiv.org/abs/2107.03374
- Distinct-n：Li et al., 2016, A Diversity-Promoting Objective Function for Neural Conversation Models, https://aclanthology.org/N16-1014/
- Self-BLEU 与 diversity/quality 讨论：Alihosseini et al., 2019, Jointly Measuring Diversity and Quality in Text Generation Models, https://aclanthology.org/W19-2311/
- 语义相似度 embedding/cosine 依据：Zhang et al., 2019, BERTScore, https://arxiv.org/abs/1904.09675
