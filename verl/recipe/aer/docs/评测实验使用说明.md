# AER 评测实验使用说明

## 1. 评测入口

评测有两个主入口：

| 入口 | 用途 |
|---|---|
| `eval/eval_from_jsonl.py` | 直接读取已有 validation/rollout JSONL 计算指标 |
| `eval/eval_from_model.py` | 指定模型路径，重新推理、验证、再计算指标 |

旧入口 `eval/evaluate_aer.py validation` 仍可用，但推荐新实验直接使用两个主入口。

## 2. 指标选择

`--metrics` 支持：

| 指标名 | 输出列 |
|---|---|
| `pass@k` | `pass@1/pass@2/...`、`correct_rate` |
| `first@1` | `first@1` |
| `distinct-2` | `distinct_2` |
| `self-bleu` | `self_bleu4` |
| `semantic-cosine` | `semantic_cosine` |
| `all` | 默认全测 |

示例：

```bash
--metrics pass@k,first@1
--metrics distinct-2,self-bleu
--metrics all
```

## 3. 从已有 JSONL 评测

训练时建议保留 validation JSONL：

```bash
validation_data_dir="../../../save/validation/${experiment_name}"
```

评测命令：

```bash
cd verl/recipe/aer
/data/ruanruihan/.conda/envs/aer/bin/python eval/eval_from_jsonl.py \
  --input ../../../save/validation/<experiment_name> \
  --output-dir ../../../save/eval/<experiment_name>/jsonl \
  --metrics all \
  --ks 1,2,4,8 \
  --semantic-model /data/models/Qwen/Qwen3-Embedding-0.6B \
  --semantic-device cpu \
  --semantic-batch-size 32
```

只评测 Pass@K：

```bash
/data/ruanruihan/.conda/envs/aer/bin/python eval/eval_from_jsonl.py \
  --input ../../../save/validation/<experiment_name> \
  --output-dir ../../../save/eval/<experiment_name>/jsonl_pass \
  --metrics pass@k,first@1 \
  --ks 1,2,4,8
```

旧 JSONL 若没有 `data_source` 字段，会统一归为 `all`，无法得到分数据集结果。

## 4. 从模型重新推理评测

用于以下场景：

1. validation JSONL 没有保存。
2. 需要更大的 Pass@K，例如 16/32/64。
3. 希望所有实验使用完全相同的验证采样参数。

如果目标 checkpoint 是 FSDP 分片格式，先合并为 HuggingFace 模型：

```bash
cd verl
/data/ruanruihan/.conda/envs/aer/bin/python scripts/model_merger.py merge \
  --backend fsdp \
  --local_dir ../save/checkpoints/<experiment_name>/global_step_<S>/actor \
  --target_dir ../save/merged_hf/<experiment_name>/global_step_<S>
```

重新推理并评测：

```bash
cd verl/recipe/aer
/data/ruanruihan/.conda/envs/aer/bin/python eval/eval_from_model.py \
  --model-path ../../../save/merged_hf/<experiment_name>/global_step_<S> \
  --val-files ../../../save/data/math-ai/math500/test_repeated.parquet \
              ../../../save/data/math-ai/amc23/test_repeated.parquet \
              ../../../save/data/math-ai/aime24/test_repeated.parquet \
              ../../../save/data/math-ai/aime25/test_repeated.parquet \
  --output-dir ../../../save/eval/<experiment_name>/rerun_pass32 \
  --metrics pass@k,first@1,distinct-2,self-bleu \
  --ks 1,2,4,8,16,32 \
  --backend vllm \
  --tensor-parallel-size 4 \
  --temperature 0.6 \
  --top-p 0.95 \
  --top-k 20 \
  --max-new-tokens 4096 \
  --seed 42
```

推理次数规则：

- `--metrics` 包含 `pass@k`：每题生成 `max(--ks)` 次。
- `--metrics` 不包含 `pass@k`：每题只生成 1 次。
- 需要固定其它样本数时，显式传 `--samples-per-prompt N`。

当前 `test_repeated.parquet` 的 repeat 次数为 1。脚本默认按 `unique_id`、`extra_info.index` 或 prompt 去重；若后续输入真正重复的 parquet，默认会去重。若确实想保留重复样本，加 `--no-deduplicate`。

## 5. 解析训练日志

```bash
/data/ruanruihan/.conda/envs/aer/bin/python eval/evaluate_aer.py train-log \
  --input log.txt \
  --output-dir ../../../save/eval/<experiment_name>/train_log
```

默认导出：

- `metric/exploration reward`
- `metric/weight`
- `metric/acc reward`
- `actor/entropy_loss`
- `response_length/clip_ratio`
- `response_length/mean`

导出所有数值指标：

```bash
/data/ruanruihan/.conda/envs/aer/bin/python eval/evaluate_aer.py train-log \
  --input log.txt \
  --output-dir ../../../save/eval/<experiment_name>/train_log \
  --all-keys
```

## 6. 生成 tau 候选值

```bash
/data/ruanruihan/.conda/envs/aer/bin/python eval/evaluate_aer.py tau-plan \
  --input log.txt \
  --algorithm <similarity_algorithm> \
  --output ../../../save/eval/tau_plan_<similarity_algorithm>.csv
```

输出：

- `E_early`
- `E_mid`
- `E_late`
- `E_floor`
- `tau_anchor`
- `tau_low`
- `tau_mid`
- `tau_high`

## 7. 输出文件

| 文件 | 内容 |
|---|---|
| `validation_summary.csv` | 按 step 和数据集聚合的指标 |
| `validation_summary.json` | summary JSON |
| `validation_summary.md` | 可直接复制到实验记录的 Markdown 表格 |
| `validation_per_prompt.csv` | 每道题的 per-prompt 指标 |
| `model_rollout.jsonl` | 仅 `eval_from_model.py` 生成，保存重新推理的逐样本输出 |
| `model_rollout.meta.json` | 仅 `eval_from_model.py` 生成，保存推理参数 |

## 8. 注意事项

1. 比较不同实验时，验证集、采样参数、`--ks` 和 `--metrics` 必须一致。
2. `semantic-cosine` 依赖 `sentence-transformers` 和 embedding 模型；请求该指标时必须传 `--semantic-model`。
3. 若每题只有 1 个输出，Self-BLEU 和 Semantic Cosine 为空；Distinct-2 仍可计算单输出内部 2-gram 多样性。
4. `eval_from_model.py` 的 `--backend auto` 会优先尝试 vLLM，失败后使用 HuggingFace。
