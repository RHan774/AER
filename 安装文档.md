# 安装文档

## 直接开始（环境准备好后）

### 开始训练

```bash
./run.sh
```

### 中断训练

```bash
pkill -f main_ppo
```

### 中断后恢复训练

```bash
# 方式1：自动恢复（推荐） - 默认配置已启用
# run.sh 中 resume_mode: auto（默认）
# 会自动从最新checkpoint恢复
bash run.sh

# 方式2：指定checkpoint恢复
# 在 run.sh 中添加
trainer.resume_from_path="checkpoints/AER/aer-tau0.4/global_step272"
# 会从 global_step272 开始恢复训练
bash run.sh
```

验证恢复是否成功：查看日志中是否有

```bash
Checkpoint tracker file exists: ...
Resuming from checkpoint: global_stepXXX
```

1. 创建环境

```bash
conda create -n aer python=3.10 -y

```

## 遇到的问题及解决

### Ray临时目录问题 -已经改了run.sh可以不用管这里

> 报错`requests.exceptions.HTTPError: 502 Server Error: Bad Gateway for url: http://127.0.0.1:8265/api/version`
> 发现是因为 Ray 的临时目录（ /tmp/ray ）中保留了旧的会话信息，导致新的会话启动时冲突。可以删除Ray临时目录：`rm -rf /tmp/ray`。
> **没有root权限的方法：**

```bash
# 在您的 home 目录下创建一个临时目录
mkdir -p ~/ray_tmp

# 设置环境变量
export RAY_TMPDIR=~/ray_tmp

# # 然后可以启动 Ray（run.sh修改后，用本地Ray集群不需要启动）
# ray start --head --num-gpus=4
```
